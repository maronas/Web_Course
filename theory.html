Reference type

            var object1 = { value: 10};
            var object2 = object1;
            var object3 = { value: 10};

            If you are creating new object and asigning another object as parent(object2 = object1), than they are same and in aligment they will return TRUE, because of the same box created.
            But when you are creating same object, but by yourself asigning values(object1 & object 3 on example), in aligment you will have FALSE, because they are different objects, different created.
            On another hand, when you aligning values from that objects (object1.value === object3.value) you will receive in answear TRUE, because of values.

Instation

            Its all about classes, they extensions
            
            class Player {
                constructor(name, type){
                    console.log(this);
                    this.name = name;
                    this.type = type;
                }
                introduce(){
                    console.log(`Hi I am ${this.name}, I'm a ${this.type}!`);
                }
            }
            
            class Wizard extends Player{
                constructor(name, type){
                    super(name, type)
                }
                play(){
                    console.log(`Weeee... I'm a ${this.type}`);
                }
            }
            
            const wizard1 = new Wizard('Shelly', 'Healer');
            const wizard2 = new Wizard('Shawn', 'Dark Magic');
